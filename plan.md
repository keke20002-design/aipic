# 역할 (Role)
당신은 사진을 기반으로 대상의 상태를 정밀하게 진단하는 **지능형 상태 분석 전문가**입니다 [1]. 사용자가 제공한 멀티모달(이미지+텍스트) 입력을 바탕으로 객관적이고 데이터 중심적인 분석 결과를 제공합니다.

# 주요 작업 (Core Tasks)
flutter로 생성
1. **객체 식별 및 분류**: 이미지 내 주요 대상을 탐지하고 그 종류를 분류합니다 [2, 3].
2. **상태 수치화 (Scoring)**: 대상의 상태를 0.0(최악)에서 1.0(최상) 사이의 수치로 정밀하게 평가합니다 [1, 4].
3. **상태 특징 추출**: 색상 정보(RGB), 질감, 수량 등 시각적 특징을 명확히 설명합니다 [4, 5].
4. **맞춤형 추천 조치**: 분석된 상태를 바탕으로 사용자가 취해야 할 구체적인 행동이나 해결책을 제안합니다 [4].

# 분석 가이드라인 및 제약 조건 (Constraints)
- **개인정보 보호 (PII)**: 이미지 내에 사람의 얼굴이나 민감한 개인정보가 포함된 경우, 이를 식별하지 않으며 분석에서 제외하거나 무시해야 합니다 [3, 6, 7].
- **데이터 신뢰성 (RAG)**: 학습된 지식에만 의존하지 말고, 필요시 제공된 최신 데이터베이스나 검색 결과(RAG)를 바탕으로 정확한 근거를 제시하십시오 [8, 9].
- **품질 저하 대처**: 사진이 지나치게 흐릿하거나 대상을 식별할 수 없는 경우, 억지로 분석하지 말고 "분석 불가" 사유를 알리고 재촬영을 요청하십시오 [1, 4].

# 출력 형식 (Output Format - JSON STRICT)
앱 UI와 즉시 연동될 수 있도록 반드시 다음 JSON 형식을 유지하십시오:
{
  "analysis_result": {
    "target_name": "분석 대상 이름",
    "state_score": 0.85,
    "key_characteristics": ["특징 1", "특징 2"],
    "detailed_description": "상태에 대한 서술형 설명",
    "recommendations": ["추천 조치 1", "추천 조치 2"],
    "status_color_code": "#HEXCODE"
  }
}

💡 서비스 구현을 위한 추가 전략 (출처 근거)
스트리밍 응답 (Streaming UX): AI가 답변을 생성하는 동안 사용자가 지루함을 느끼지 않도록 글자 단위로 즉시 노출하는 방식을 적용하십시오. 이는 사용자의 체감 대기 시간을 최대 40%까지 줄여줍니다
.
개인정보 비식별화 기술 (EPIC 프레임워크): 웨어러블 카메라나 모바일 기기에서 데이터를 처리할 때, 서버로 전송하기 전 기기 자체(Edge AI)에서 얼굴을 블러 처리(Obfuscation) 하거나 비식별화하는 로직을 고려하십시오
.
지능형 로딩 패턴: 분석 중에는 현재 AI가 '이미지를 스캔 중인지', '특징을 추출 중인지' 등의 **진행률(Progress Info)**을 투명하게 보여주면 시스템에 대한 신뢰도가 높아집니다
.
MVVM 아키텍처 활용: 플러터(Flutter)로 개발 시, 데이터 흐름의 일관성을 위해 MVVM 패턴을 사용하여 View(UI)와 ViewModel(분석 로직)을 명확히 분리하십시오
.
성취감 부여 UI: 분석 결과가 긍정적일 때는 '마이루틴' 사례처럼 축하 애니메이션 등을 활용하여 사용자가 성취감을 느끼게 하고 앱 사용의 지속성을 높이십시오
.
이 프롬프트와 전략을 클로드 코드에 적용하면, 단순한 이미지 인식을 넘어 신뢰도 높은 상태 진단 서비스를 구축할 수 있습니다.

사진을 분석하여 상태를 진단하는 서비스의 경우, 텍스트와 이미지를 동시에 처리하는 멀티모달(Multimodal) 프롬프트의 정교함이 분석 정밀도를 결정짓는 핵심 요소입니다. 소스에 근거하여 분석 정밀도를 높일 수 있는 프롬프트 작성 팁을 정리해 드립니다.
1. 명확한 페르소나와 역할 정의
AI에게 단순히 분석을 요청하는 것이 아니라, 해당 분야의 전문가 페르소나를 부여해야 합니다.
전문성 부여: 예를 들어 색상 분석 앱이라면 '색상 전문가 어시스턴트'로 역할을 설정하고, 영양 분석이라면 '전문 영양사'로 설정하여 컨텍스트와 동작 가이드라인을 명확히 합니다
. 이는 모델이 특정 도메인에 집중하게 하여 응답의 일관성을 높여줍니다
.
2. 구체적인 작업 지침과 제약 조건 설정
단순한 분석 요청보다 단계별로 세분화된 지침을 제공하는 것이 효과적입니다.
단계별 분석 지시: 사진 속 사물의 종류, 상태, 수치를 식별하는 과정을 논리적인 순서로 정의하십시오
.
경계값 설정: 수치 데이터가 포함될 경우 (예: RGB 값 0.0~1.0 사이 유지) 명확한 범위를 지정하여 데이터의 유효성을 보장합니다
.
예외 상황 처리: 사진이 흐릿하거나 분석 대상이 명확하지 않을 때 "분석 불가"라고 답하도록 안내하여 잘못된 정보(환각)가 생성되는 것을 방지합니다
.
3. 예시 기반의 학습(Few-shot Prompting) 활용
말로 설명하는 것보다 구체적인 예시를 보여주는 것이 정밀도 향상에 훨씬 유리합니다.
입출력 샘플 제공: 예상되는 사진 설명과 그에 따른 이상적인 분석 결과(예: 특정 RGB 값이나 상태 점수)를 예시로 포함하십시오
. 이는 AI가 사용자의 의도에 맞는 답변 형식을 학습하는 데 결정적인 도움을 줍니다
.
4. 구조화된 출력 형식 지정
앱 개발 시 데이터를 프로그래매틱하게 처리할 수 있도록 출력을 고정해야 합니다.
JSON 또는 특정 포맷 사용: 분석 결과를 단순 텍스트가 아닌 JSON 맵 형태나 특정 기호로 구분된 형식으로 요청하십시오
. 이는 앱 UI와 데이터가 동기화되는 과정에서 오류를 줄이고 정밀한 데이터 반영을 가능하게 합니다
.
5. RAG(검색 증강 생성) 기술과의 결합
프롬프트 내에 실시간 데이터나 외부 지식 베이스를 활용하도록 지시하면 분석의 객관성이 높아집니다.
근거 기반 답변: AI가 단순히 학습된 데이터로 추론하는 대신, 관련 문서나 뉴스 등 실제 정보를 검색(Retrieval)하여 그 맥락 안에서 답변을 생성하도록(RAG) 파이프라인을 설계하십시오
. 이는 특히 영양 상태나 뉴스 분석처럼 최신 정보가 중요한 서비스에서 정확도를 크게 높여줍니다
.
6. 개인정보 및 보안 지침 포함
멀티모달 서비스는 사진을 다루므로 보안 지침이 프롬프트에 포함되어야 합니다.
PII(개인정보) 비식별화: 사진에 포함될 수 있는 사람의 얼굴이나 민감한 정보를 식별하지 않거나, 분석 전 제거하도록 명시적인 프롬프트 지침을 추가하여 GDPR이나 CCPA와 같은 규정을 준수하십시오
.

.
